Hạng mục Anh Tuấn Đạt
Hình thức & thành phần
  • Container trên hạ tầng KH / Viettel · cụm HA theo tầng
  • HEC · Authorizer · WebUI · Context · Agent (Scheduler / Vee)
  • Playbook báo cáo điều hành 1022 (6 module)
  • Container trên 01 workstation · dự phòng khi cần
  • Next.js · FastAPI · Worker · MySQL · PostgreSQL/pgvector · Ollama
  • Dashboard · trích xuất LLM · embedding · sinh báo cáo
Hạ tầng compute
  • 6 node HA: 2 Engine + 2 DB/Context + 2 Web
  • ~16 vCPU / 32 GB RAM / ~512 GB SSD
  • Phụ trợ: LB, switch, firewall/WAF, NAS, bastion, UPS
  • 1 workstation: 12–16 nhân · RAM 64 GB · NVMe 2 TB
  • GPU RTX 4090 24 GB (hoặc tương đương)
  • NAS 2 khay + UPS ~1.500 VA (nếu chưa có sẵn)
AI / model
  • LLM On-Prem nguồn mở (Qwen / DeepSeek / Llama / Gemma)
  • Tier 1: 1× GPU 48–80 GB · Tier 2: 2× GPU 80 GB
  • Fast ~7–14B · Deep ~32B–70B (nén)
  • Ollama: ornith:35b (~21 GB) — trích xuất / báo cáo
  • qwen3-embedding:8b (~4,7 GB) — embedding 1.024 chiều
  • Model dùng tuần tự trên cùng GPU 24 GB
Quy mô / lộ trình
  • GĐ1 lãnh đạo TP → GĐ2 sở/ngành → GĐ3 cán bộ
  • Tier 1 ~10–50 lãnh đạo; Tier 2 ~150–300 (ước tính)
  • ~4 phản ánh/phút → ~12,5 giờ / đợt 3.000
  • Giai đoạn đầu: một máy dùng chung là đủ
